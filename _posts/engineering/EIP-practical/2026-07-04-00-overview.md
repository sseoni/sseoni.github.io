## 💡 “프로그래밍” 이란

현실의 불편함이나 요구사항을 하나의 완성된 프로그램으로 해결하는 것입니다. 단순히 컴퓨터 앞에 앉아 코드를 작성하는 행위(코딩)뿐만 아니라, **기획부터 설계, 제작, 검수, 출시까지의 모든 과정**을 통틀어 프로그래밍이라고 부릅니다.

정보처리기사의 과목들은 이 '소프트웨어가 탄생하는 처음과 끝(라이프사이클)'을 그대로 담고 있습니다. 우리가 흔히 쓰는 '배달 앱'을 처음부터 만든다고 가정하며 정처기 과목들의 흐름을 따라가 봅시다.

<br>
### 🏗️ 1단계: 기획과 설계 (어떤 건물을 지을까?)

가장 먼저 앱의 목적을 정하고 밑그림을 그리는 단계입니다.

| 개발 단계 | 세부 내용 | 필기 / 실기 범위 |
| --- | --- | --- |
| 개발 환경 파악 | 앱을 24시간 띄워놓을 서버(리눅스/윈도우)와 데이터가 오갈 통신망<br>(TCP/IP) 환경을 파악합니다. | 응용 SW 기초 기술 활용 <br> 필기 4과목 / 실기 11단원 |
| 요구사항 수집 | 클라이언트가 "배달 앱에 찜하기 기능이랑 리뷰 기능 꼭 넣어주세요!"<br>라고 하는 구체적인 요구사항을 확인합니다. | 요구사항 확인 <br> 필기 1과목 / 실기 1단원 |
| 화면 디자인 | 사용자가 앱을 켰을 때 볼 화면의 버튼 위치 등을 디자인합니다. | 화면 설계 <br> 필기 1과목 / 실기 6단원 |
| 데이터 설계 | 수만 개의 식당 메뉴와 가격 정보를 어떻게 정리해서 저장할지 데이터의 <br>뼈대를 조직화합니다. | 데이터 입출력 구현 <br> 필기 3과목 / 실기 2단원 |

<br>
### 🛠️ 2단계: 본격적인 개발과 조립 (철골을 세우고 파이프 연결하기)

설계도를 바탕으로 개발 언어를 사용해 진짜 작동하는 앱을 만듭니다.

| 개발 단계 | 세부 내용 | 필기 / 실기 범위 |
| --- | --- | --- |
| 언어 선택 및 코딩 | Java, C, Python 등 프로젝트에 맞는 언어를 선택하고 핵심 로직을<br>코딩합니다. | 프로그래밍 언어 활용 <br> 필기 4과목 / 실기 10단원 |
| 서버 개발 | "회원가입 처리", "주문 접수" 등 앱의 심장 역할을 하는 백엔드(서버)<br>로직을 개발합니다. | 요구사항 확인 <br> 필기 2과목 / 실기 4단원 |
| 데이터베이스 조작 | 데이터베이스에 접근해 '오늘의 주문 내역'을 저장하고 불러오기 위한 <br>쿼리문을 작성합니다. | SQL 응용 <br> 필기 3과목 / 실기 8단원 ] |
| 외부 시스템 연동 | 카카오페이나 토스 같은 외부 결제 시스템과 우리 앱이 매끄럽게 통신<br>하도록 연결합니다. | 통합 구현 [ 실기 3단원 ] <br> 인터페이스 구현 [ 실기 5단원 ] <br> 필기 2과목 / 실기 3, 5단원 |
| 보안 세팅 | 해커가 고객의 신용카드 정보를 빼가지 못하도록 암호화하고 보안<br>취약점을 차단합니다. | 소프트웨어 개발 보안 구축 <br> 필기 5과목 / 실기 9단원 |

<br>
### 🚀 3단계: 검수 및 출시 (안전 점검과 오픈)

완성된 앱을 대중에게 공개하기 전, 마지막 단장을 하는 단계입니다.

| 개발 단계 | 세부 내용 | 필기 / 실기 범위 |
| --- | --- | --- |
| 테스트 | 결제 도중에 앱이 튕기지는 않는지, 리뷰가 제대로 안 올라가는 버그는<br>없는지 철저하게 테스트합니다. | 애플리케이션 테스트 관리 <br> 필기 2과목 / 실기 7단원 |
| 출시 및 배포 | 모든 테스트를 통과한 앱을 사용자들이 구글 플레이스토어나 앱스토어<br>에서 다운받을 수 있도록 예쁘게 압축하고 포장합니다.<br>설치와 사용에 필요한 환경 등의 메뉴얼을 작성하여 고객에게 전달하는 과정도 패키징에 해당합니다. | 제품 소프트웨어 패키징 <br> 필기 2과목 / 실기 12단원 |
