## 7️⃣ 응용계층
**사용자에게 실제 네트워크 서비스 제공**
: 사용자가 직접 눈으로 보고 다루는 프로그램(웹 브라우저, 메일 클라이언트 등)이 돌아가는 구역
`게이트웨이(Gateway)`를 사용해 **서로 다른 프로토콜망을 연결**한다.
<br>

| 프로토콜 | 역할 | Port Number |
| --- | --- | --- |
| **HTTP**<br>**H**yper **T**ext **T**ransfer **P**rotocol | 웹에서 HTML 문서를 송수신 | 80번 포트 |
| **FTP**<br>**F**ile **T**ransfer **P**rotocol | 파일 전송 | 20, 21번 포트 |
| **SMTP**<br>**S**imple **M**ail **T**ransfer **P**rotocol | 이메일 송신 | 25번 포트 |
| **POP3 / IMAP** | 이메일 수신 | |
| **DHCP**| 단말기에 IP 주소를 동적(자동)으로 할당하고 사용이 끝나면 회수 | |
| **DNS**<br>**D**omain **N**ame **S**ystem | 도메인 이름(예: google.com)을 IP 주소(숫자)로 변환 | |
| **SSH**<br>**S**ecure **Sh**ell | 원격지 컴퓨터에 안전하게 접속하기 위해 사용되는 암호화 프로토콜<br>(보안이 취약한 Telnet을 대체) | 22번 포트 |

<br>

## 6️⃣ 표현계층
데이터 형식 변환, 압축, **암호화/복호화** 수행

<br>

## 5️⃣ 세션계층
송수신 프로세스 간 **연결 유지 및 동기화** 처리
- **동기점 (Checkpoint)**: 데이터를 전송하다가 연결이 끊기면 처음부터 다시 받지 않도록 중간중간 '저장 포인트(동기점)'를 찍어둔다.
- **대화 제어**: 데이터를 번갈아 보낼지, 동시에 보낼지 관리
- **세션 연결 / 유지 / 종료**: 응용 프로그램 간의 논리적인 연결을 담당 (대표 프로토콜: RPC)

<br>

## 4️⃣ 전송계층
전송계층에서는 **데이터를 어떻게 보낼지** 규정: 데이터를 세그먼트로 자르고, 번호 붙임 ➡️ 수신확인 ➡️ 흐름제어, 혼잡제어

종단 간(End-to-End) **신뢰성 있는 데이터 통신** 보장 (TCP/UDP)
`L4 스위치`로 **트래픽 로드밸런싱**

| 프로토콜 | 역할 |
| --- | --- |
| **TCP**<br>(Transmission Control Protocol) | **연결 지향**: 통신 전 3-Way Handshake로 미리 연결<br>**신뢰성** 보장: 패킷의 손실/중복을 막고 순서대로 잘 도착하게함<br>  → 순서 제어, 흐름 제어, 혼잡 제어 기능 제공  |
| **UDP**<br>(User Datagram Protocol) | 비연결형, 신뢰성은 낮으나 **전송 속도**가 매우 빠름<br>→ 확인 응답이나 오버헤드가 적어 유튜브 등 실시간 스트리밍에 주로 사용 |

<br>

## 3️⃣ 네트워크 계층
네트워크 계층에서는 **데이터를 어떤 길로 보낼지** 규정: 전송 주소(IP), 최적경로(라우팅)

데이터를 전송하는 기본 단위: **패킷 (데이터 + IP헤더)**
`L3 스위치`를 통해 **논리적 주소(IP) 지정**
`라우터(Router)`로 **최적 경로 설정(라우팅)**

#### 💡 IP 주소체계 `IPv4`, `IPv6`

| 구분 | IPv4 | IPv6 |
| -- | -- | -- |
| 길이 | 32비트 (8비트씩 4부분) | 128비트 (16비트씩 8부분) |
| 할당 방식 | 유니캐스트, 멀티캐스트, **브로드캐스트** | 유니캐스트, 멀티캐스트, **애니캐스트(Anycast)** |

- **[ IPv4 ➡️ IPv6 전환 기술 ]**
- 듀얼 스택(Dual Stack): 기기에 두 통신 프로토콜을 모두 탑재
- 터널링(Tunneling): IPv6 패킷을 캡슐화하여 IPv4 망을 통과시킴
- 헤더 변환(Header Translation): IP 헤더 자체를 서로 변환하여 통신

#### 💡 라우팅(Routing) 프로토콜 (3계층 경로 탐색 알고리즘)
**정적(Static) 라우팅**: 관리자가 수동으로 경로를 직접 설정
**동적(Dynamic) 라우팅**: 라우터끼리 정보를 교환하여 스스로 최적 경로 갱신


| 프로토콜 | 특징 | 알고리즘 / 용도 |
| -- | -- | -- |
| RIP | 거리 벡터(Distance Vector) 알고리즘 기반<br>최대 홉(Hop 거치는 라우터 수)을 15개로 제한 | 소규모 망<br>벨만-포드 알고리즘 |
| OSPF | 링크 상태(Link State) 알고리즘 기반<br>도로의 막힘 정도 등 링크 상태를 파악해 최적 경로 계산 | 대규모 네트워크<br>다익스트라 알고리즘 |
| BGP | 경로 벡터(Path Vector) 알고리즘 기반 | AS(자율 시스템) 간 라우팅(외부 라우팅, EGP) |


#### 💡 3계층 주요 프로토콜

| 프로토콜 | 역할 |
| -- | -- |
| IP | 목적지 주소(경로)를 찾아서 패킷을 전달 (경로 탐색, 비신뢰성) |
| ICMP | 인터넷 제어 메시지 프로토콜. 통신 중 오류가 발생하면 에러 메시지를 알려줌 (예: ping 명령어) |
| ARP | 주소 분해 프로토콜. **IP 주소(논리적 주소)**를 **MAC 주소(기계의 물리적 주소)**로 변환<br>cf. 반대로 MAC을 IP로 바꾸는 건 RARP |

```
※ MAC 주소란?
네트워크 인터페이스 카드(NIC)에 할당된 48비트의 물리적 고유 주소 (변경 불가)
```

<br>

## 2️⃣ 데이터 링크 계층
인접 노드 간 신뢰성 있는 전송, MAC 주소 제어, 에러 검출
주요 장비: `스위치(Switch)`, `브리지(Bridge)`
대표 프로토콜: `HDLC (High-level Data Link Control)`

<br>

## 1️⃣ 물리 계층
전기적 신호로 변환하여 전송
주요 장비: `허브(Hub)`, `리피터(Repeater)`로 신호를 증폭하여 데이터 전송
대표 프로토콜: `X.21`
