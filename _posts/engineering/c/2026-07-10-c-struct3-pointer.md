구조체 안에 문자열(이름, 상태 등)을 저장할 때, C언어에서는 두 가지 방법을 쓸 수 있습니다. 하나는 `char 배열`이고, 다른 하나는 `char 포인터`입니다. 

두 방식은 겉보기엔 비슷하지만, **'나중에 값을 변경할 때'** 완전히 다르게 동작합니다. 출제자들이 오답을 유도하기 위해 가장 사랑하는 이 함정을 파헤쳐 봅시다.

## 1. 함정 비교 실습 코드

```c
#include <stdio.h>
#include <string.h> // strcpy를 사용하기 위한 헤더

typedef struct {
    char nameArray[20];   // 🚨 방식 1: 고정된 문자열 배열
    char *namePointer;    // 🟢 방식 2: 문자열 포인터
} Profile;

int main() {
    // 초기화는 두 방식 모두 동일하게 작동합니다.
    Profile p = {"Kim", "Lee"};

    // ============================================
    // 🚨 [핵심 함정] 값(문자열) 변경 시도!
    // ============================================

    // ✅ 성공: 포인터는 '주소'를 담는 방이므로, 새로운 문자열의 주소로 자유롭게 갈아탈 수 있습니다.
    p.namePointer = "Park"; 
    printf("포인터 변경 성공: %s\n", p.namePointer);

    // ❌ 실패(컴파일 에러): 배열은 이미 크기가 고정된 메모리 공간입니다. 대입 연산자(=)로 덮어씌울 수 없습니다.
    // p.nameArray = "Choi"; 

    // 🟢 배열의 글자를 바꾸려면 반드시 strcpy(문자열 복사) 함수를 써야 합니다.
    strcpy(p.nameArray, "Choi");
    printf("배열 변경 성공(strcpy): %s\n", p.nameArray);

    return 0;
}
```

## 2. 왜 이런 차이가 발생할까? (원리 이해하기)

### 📌 `char *namePointer` (자유로운 세입자)

포인터는 단지 '주소값'을 저장하는 8바이트짜리 작은 방입니다.
처음엔 `"Lee"`가 있는 주소를 가리키다가, `p.namePointer = "Park";` 명령을 받으면 기존 주소를 버리고 `"Park"`이 있는 새로운 주소로 손가락 방향만 쿨하게 바꿔버립니다.

### 📌 `char nameArray[20]` (철거 불가능한 내 집)

배열을 선언하는 순간, 메모리에 20칸짜리 공간이 단단하게 시멘트로 굳어집니다.
`p.nameArray = "Choi";` 처럼 대입 연산자(`=`)를 쓴다는 것은, 이미 지어진 20칸짜리 집을 통째로 들어내고 새로운 집을 욱여넣으려는 시도와 같습니다. 컴파일러가 이를 허락하지 않습니다.
따라서, 기존 집은 그대로 둔 채 안에 있는 글자(가구)만 하나씩 새로 채워 넣어야 하며, 이 역할을 해주는 것이 바로 `strcpy` 함수입니다.

### 💡 요약

구조체 안에 `*` (포인터)로 된 문자열이 있다 ➔ `=` 대입 연산자로 값 변경 가능 🟢
구조체 안에 `[]` (배열)로 된 문자열이 있다 ➔ `=` 변경 불가능 🔴, 반드시 `strcpy` 사용!
