지난 포스트에서 `%c`와 `%s`의 기본 개념을 정리했습니다.
이번에는 정보처리기사 실기에 자주 나오는 **'포인터 변수와 서식 지정자의 매칭'** 문제를 다뤄보겠습니다.

컴파일 경고(Warning)나 실행 오류를 유발하는 원인으로 정보처리기사 실기 시험의 단골 함정이니 확실하게 살펴봅시다.

## 1. 흔히 하는 실수 (Warning 발생 코드)

아래 코드는 컴파일 시 `-Wformat` 경고가 발생하거나 실행 시 강제 종료될 수 있는 잘못된 코드입니다.

```c
#include <stdio.h>

int main() {
    char *p = "BANANA"; // 문자열의 시작 주소를 포인터 p에 저장

    // [에러 발생 지점] %s는 주소를 원하는데, *p는 실제 값(문자 1개)을 줍니다.
    printf("%s\n", *p); 

    return 0;
}
```

### 왜 안될까? (핵심 원리)

**`%s` 변환 기호의 본질**: 문자열을 기대

- 문자열을 출력하는 `%s`는 "어느 주소부터 글자를 읽기 시작할까?" 라며 메모리의 시작 주소(위치값)를 기다립니다.

**`*p` 연산자의 본질**: 주소 `p`에 담긴 글자 하나

- 포인터 앞의 `*`(역참조 기호)는 주소의 껍데기를 까서 그 안에 들어있는 '실제 값'을 꺼냅니다. 즉, `*p`는 "BANANA" 전체가 아니라 첫 번째 글자인 단일 문자 'B'만을 의미합니다.

**결과**: 주소를 받아야 하는 `%s` 자리에 달랑 문자 1개(`char`)인 'B'를 던져주었기 때문에 컴파일러가 타입 불일치 오류를 뱉어내게 됩니다.

## 2. 올바른 해결 공식 2가지

화면에 출력하고자 하는 최종 목적에 따라 코드를 다르게 수정해야 합니다.

### 방법 A: 단어 전체("BANANA")를 다 출력하고 싶을 때

`%s`가 원하는 대로 주소값 자체를 담고 있는 포인터 변수 `p`를 그대로 넘겨줍니다.

```C
#include <stdio.h>

int main() {
    char *p = "BANANA";

    // p는 주소값이므로 %s와 완벽하게 짝꿍이 맞습니다.
    printf("%s\n", p); // 출력 결과: BANANA

    return 0;
}
```

### 방법 B: 딱 첫 글자('B') 한 개만 출력하고 싶을 때

`*p`로 문자 1개를 꺼냈다면, 출력 기호도 문자 1개용인 `%c`로 매칭해 주어야 합니다.

```C
#include <stdio.h>

int main() {
    char *p = "BANANA";

    // *p는 단일 문자이므로 %c와 완벽하게 짝꿍이 맞습니다.
    printf("%c\n", *p); // 출력 결과: B

    return 0;
}
```

## 3. 정처기 시험용 암기 매칭 공식

`%s + p` (포인터 변수 이름 그대로) = 문자열 전체 출력
`%c + *p` (포인터가 가리키는 실제 값) = 문자 딱 1개 출력
