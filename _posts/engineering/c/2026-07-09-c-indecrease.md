정보처리기사 실기 C언어 문제에서 수험생들을 가장 많이 낚는 두 가지 함정은 바로 **'증감 연산자의 위치(전위/후위)'**와 **'문자형(char)의 사칙연산'**입니다. 아래 실습 코드를 통해 출력 결과를 완벽하게 추적해 봅시다.

## 1. 실습 코드 및 정답 추적

```c
#include <stdio.h>

int main() {
    int a = 10;
    int b = 11;      // 변수 선언 및 초기화
    char c = 'a';    
    int d = 7;       

    // 1. 후위 증감 연산자 (a++)
    // 출력부터 먼저 하고(10), 그 다음 줄로 넘어갈 때 a를 1 증가시킴 (a는 11이 됨)
    printf("%d\n", a++); 
    
    // 2. 전위 증감 연산자 (++b)
    // b를 먼저 1 증가시키고(12), 그 증가된 값을 출력함
    printf("%d\n", ++b); 
    
    // 3. 아스키코드 문자 연산 (c + 1)
    // 소문자 'a'의 아스키코드 값은 97. 97 + 1 = 98이므로 문자 'b' 출력
    printf("%c\n", c + 1);
    
    // 4. 아스키코드 대소문자 변환 (c - 32)
    // 97 - 32 = 65. 아스키코드 65는 대문자 'A'
    printf("%c\n", c - 32);
    
    // 5. 변경된 변수 확인
    // 1번 줄에서 후위 증감으로 인해 11이 된 a의 값 출력
    printf("%d\n", a);
    
    // 6. 정수형 나눗셈
    // 7 / 2 = 3.5 이지만, 정수(int)끼리의 연산이므로 소수점 이하는 버려짐(3 출력)
    printf("%d\n", d / 2); 

    return 0;
}
```

## 2. 코드 설명 및 시험장 유의사항

### 📌 핵심 1: 증감 연산자 (전위 vs 후위)

**`a++` (후위)**: "일단 쓰고 나중에 올려라" 현재 줄에서는 원래 값(10)이 사용되고, 다음 줄로 넘어갈 때 11이 됩니다.
**`++b` (전위)**: "일단 올리고 나서 써라" 현재 줄에서 바로 1을 더한 값(12)이 사용됩니다.

### 📌 핵심 2: 문자형(char)의 아스키코드 연산

문자에 숫자를 더하거나 빼는 경우 아스키 코드를 떠올리세요. C언어에서 문자는 내부적으로 숫자(아스키코드)로 처리됩니다.

소문자 `a` (97) ~ `z` (122)
대문자 `A` (65) ~ `Z` (90)

**대소문자 변환 공식**: 소문자에서 32를 빼면 대문자가 되고, 대문자에서 32를 더하면 소문자가 됩니다. (예: c - 32)

### 📌 핵심 3: 정수형 나눗셈의 함정

C언어에서 정수 / 정수의 결과는 무조건 정수입니다. 7 / 2는 3.5가 아니라, 소수점을 버린 3이 출력된다는 점을 잊지 마세요!
