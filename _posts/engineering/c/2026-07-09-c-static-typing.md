C언어를 처음 접할 때 파이썬(Python)이나 자바스크립트(JS)를 쓰던 분들이 가장 답답해하는 부분이 있습니다.
바로 **"변수의 자료형(Type)이 한 번 정해지면 절대 바뀌지 않는다"**는 C언어의 고집스러운 특성입니다.

이를 전문 용어로 **정적 타입(Static Typing)** 언어라고 합니다.

## 정적 타입(Static Typing)의 엄격한 규칙

C언어에서 변수를 선언하는 것은 집을 지을 때 '방의 용도'를 설계도에 박아버리는 것과 같습니다. `int`로 만든 방에는 숫자만 들어가야지, 갑자기 문자열이 들어오면 컴파일러가 공사를 중단시켜 버립니다.

```c
#include <stdio.h>

int main() {
    // 1. 정수를 담겠다고 int형 변수 선언
    int age = 20; 
    
    // 2. 숫자를 다른 숫자로 바꾸는 건 OK
    age = 30;     
    
    // 3. 🚨 에러 발생! int 방에 문자열을 넣으려고 시도함
    // age = "서른살"; (이 코드는 컴파일조차 되지 않습니다)

    printf("나이: %d\n", age);

    return 0;
}
```

### 💡 왜 이렇게 깐깐할까?

방의 크기와 용도를 미리 100% 확정해 두기 때문에, 프로그램이 실행될 때 메모리를 얼마나 쓸지 컴퓨터가 고민할 필요가 없습니다. 덕분에 실행 속도가 압도적으로 빠르며, 코드를 실행하기도 전에 문법 오류를 잡아낼 수 있다는 강력한 장점이 있습니다.

## 강제 형변환 (Type Casting)

방의 용도를 바꿀 수 없다면, 정수와 정수를 나눠서 소수점(실수)을 얻고 싶을 때는 어떻게 해야 할까요? 여기서 형변환(Casting) 개념이 등장합니다.

```c
#include <stdio.h>

int main() {
    int a = 10;
    int b = 3;

    // 🚨 함정 1: 정수 / 정수 = 정수
    // 10 / 3 은 3.333... 이지만, 결과는 소수점을 버린 3이 나옵니다.
    float wrong_result = a / b;  
    
    // ✅ 정답: 강제 형변환 사용
    // 변수 앞에 (float)를 붙여서 a를 잠시 실수(10.0)로 변신시킵니다.
    // 실수 / 정수 = 실수가 되므로 정상적으로 3.333... 이 계산됩니다.
    float right_result = (float)a / b; 

    printf("잘못된 나눗셈 결과: %f\n", wrong_result); // 출력: 3.000000
    printf("올바른 나눗셈 결과: %f\n", right_result); // 출력: 3.333333

    return 0;
}
```

### 💡 Tip

계산식에 `(float)`나 `(int)` 처럼 괄호가 쳐진 타입이 보인다면, "아, 이 순간만큼은 이 변수를 잠시 다른 타입으로 변신시키라는 신호구나!" 라고 캐치하시면 됩니다.
