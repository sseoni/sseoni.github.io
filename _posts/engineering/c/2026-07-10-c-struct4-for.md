정보처리기사 실기 시험에서 구조체 단일 변수 하나만 달랑 던져주는 친절한 문제는 잘 나오지 않습니다. 실무와 시험 모두, 여러 사람의 데이터를 한 번에 처리하기 위해 **'구조체 배열'**을 만들고 이를 **`for`문(반복문)**과 엮어서 출제하는 것이 기본 패턴입니다.

## 1. 실습 코드: 학생 성적 합계 구하기

가장 흔하게 출제되는 '데이터 누적 합산' 로직입니다.

```c
#include <stdio.h>

// 1. 학생의 이름과 점수를 담을 구조체 설계도
struct Student {
    char name[10];
    int score;
};

int main() {
    // 2. 크기가 3인 구조체 배열 선언 및 초기화
    // {이름, 점수} 세트가 3개 연속으로 메모리에 방을 잡습니다.
    struct Student classA[3] = {
        {"Kim", 80}, 
        {"Lee", 95}, 
        {"Park", 70}
    };

    int sum = 0;

    // 3. 구조체 배열을 for문으로 순회하며 점수 합산
    for(int i = 0; i < 3; i++) {
        // classA[i]는 일반 구조체 변수이므로 점(.)을 찍어서 score를 꺼냅니다.
        sum += classA[i].score; 
    }

    // (80 + 95 + 70) = 245
    printf("총점: %d\n", sum); 

    return 0;
}
```

## 2. 실전 디버깅 (변수 추적) 요령

시험장 여백에 손으로 디버깅을 할 때는 반드시 i의 변화에 따라 구조체 배열이 어떻게 호출되는지 적어보세요.

- `i = 0` 일 때: `classA[0]`의 알맹이 호출 ➔ `"Kim"`의 점수인 80이 `sum`에 누적됨
- `i = 1` 일 때: `classA[1]`의 알맹이 호출 ➔ `"Lee"`의 점수인 95가 `sum`에 누적됨
- `i = 2` 일 때: `classA[2]`의 알맹이 호출 ➔ `"Park"`의 점수인 70이 `sum`에 누적됨

배열의 인덱스(i)와 점(.) 연산자만 헷갈리지 않으면 무조건 맞출 수 있습니다.
