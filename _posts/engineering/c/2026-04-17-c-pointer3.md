포인터와 증감 연산자(`++`)가 결합된 문제는 정보처리기사 실기의 단골문제입니다.
문자가 복잡하게 얽혀 있어 보이지만, 사실 출제자가 묻는 기준은 딱 2가지, **'괄호의 유무'**와 **'연산자의 위치'**입니다.

## 🎯 절대 헷갈리지 않는 2가지 판단 기준

1. **대상 (무엇이 증가하는가?)**
   * 괄호가 **없다** (`p`에 붙음): **'주소'**가 증가합니다. (옆 방으로 이동)
   * 괄호가 **있다** (`*p`에 붙음): **'알맹이(값)'** 자체가 증가합니다. (원래 숫자에 1 더하기)

2. **순서 (언제 증가하는가?)**
   * `++`가 **앞**에 있다 (전위): **먼저 연산**(주소를 옮기거나 값을 더한 뒤에), 마지막에 출력(반환)하라!
   * `++`가 **뒤**에 있다 (후위): 일단 지금 상태 그대로 출력(반환)부터 하고, **나중에 연산**해라!

---

| 코드 | 명칭 | 순서 (언제) | 변경 대상 (무엇을) | 쉬운 해석 (동작 순서) |
| :--- | :--- | :--- | :--- | :--- |
| **`*p++`** | 후위 주소 연산 | 뒤 (일단 출력) | `p` (주소 변경) | 현재 방 알맹이 먼저 보여주고 ➔ 옆 방으로 이동해라 |
| **`*++p`** | 전위 주소 연산 | 앞 (먼저 올림) | `p` (주소 변경) | 옆 방으로 먼저 이동한 뒤 ➔ 그 방 알맹이를 보여줘라 |
| **`(*p)++`** | 후위 값 연산 | 뒤 (일단 출력) | `*p` (값 +1) | 현재 알맹이 숫자를 먼저 보여주고 ➔ 그 알맹이 숫자에 1을 더해라 |
| **`++*p`** | 전위 값 연산 | 앞 (먼저 올림) | `*p` (값 +1) | 알맹이 숫자에 먼저 1을 더하고 ➔ 더해진 숫자를 보여줘라 |

```c
#include <stdio.h>

int main() {
    int arr[3] = {10, 20, 30};
    int *p;

    // Case 1: *p++ (값 반환 후 주소 증가)
    p = arr;
    printf("1. *p++ 출력: %d\n", *p++); // 10 출력, p는 20을 가리킴

    // Case 2: *++p (주소 증가 후 값 반환)
    p = arr; 
    printf("2. *++p 출력: %d\n", *++p); // p가 20으로 이동 후 20 출력

    // Case 3: (*p)++ (값 반환 후 원본 값 증가)
    p = arr; 
    printf("3. (*p)++ 출력: %d\n", (*p)++); // 10 출력, arr[0] 값은 11이 됨

    // Case 4: ++*p (원본 값 증가 후 반환)
    p = arr; // arr[0]은 현재 11
    printf("4. ++*p 출력: %d\n", ++*p); // arr[0] 값이 12가 된 후 12 출력

    return 0;
}
```
