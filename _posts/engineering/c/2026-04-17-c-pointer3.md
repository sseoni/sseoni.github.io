// [Post 3 Code: 포인터 증감 연산자]
#include <stdio.h>

int main() {
    int arr[3] = {10, 20, 30};
    int *p;

    // Case 1: *p++ (값 반환 후 주소 증가)
    p = arr;
    printf("1. *p++ 출력: %d\n", *p++); // 10 출력, p는 20을 가리킴

    // Case 2: *++p (주소 증가 후 값 반환)
    p = arr; 
    printf("2. *++p 출력: %d\n", *++p); // p가 20으로 이동 후 20 출력

    // Case 3: (*p)++ (값 반환 후 원본 값 증가)
    p = arr; 
    printf("3. (*p)++ 출력: %d\n", (*p)++); // 10 출력, arr[0] 값은 11이 됨

    // Case 4: ++*p (원본 값 증가 후 반환)
    p = arr; // arr[0]은 현재 11
    printf("4. ++*p 출력: %d\n", ++*p); // arr[0] 값이 12가 된 후 12 출력

    return 0;
}

시험에 자주 나오는 포인터와 증감 연산자(++)의 결합 형태를 4가지로 완벽하게 분류합니다. 해석의 기준은 **'괄호의 유무'**와 **'연산자의 위치'**입니다.

*p++ (후위 주소 연산): 현재 주소의 값을 먼저 처리하고(반환), 포인터가 다음 칸으로 이동합니다. (값은 그대로, 주소 변경)

*++p (전위 주소 연산): 포인터를 먼저 다음 칸으로 이동시킨 후, 이동한 곳의 값을 처리합니다. (값은 그대로, 주소 변경)

(*p)++ (후위 값 연산): 괄호로 인해 대상이 '값'이 됩니다. 현재 값을 먼저 처리한 후, 해당 주소에 들어있는 원본 값 자체를 1 증가시킵니다. (주소 고정, 원본 데이터 변경)

++*p (전위 값 연산): 해당 주소의 원본 값 자체를 먼저 1 증가시킨 후 그 값을 처리합니다. (주소 고정, 원본 데이터 변경)
