배열과 포인터의 관계, 그리고 연산자 우선순위에 대한 함정을 다룹니다.

```c
#include <stdio.h>

int main() {
    int ary[3] = {10, 20, 30};

    // 1. 배열명은 포인터 상수이다.
    printf("%p\n", ary);        // 배열 시작 주소
    printf("%p\n", &ary[0]);    // 첫 번째 요소 주소 (위와 동일한 주소)

    // 2. 포인터 연산과 연산자 우선순위
    int result1 = *ary + 1;    // *ary 값을 먼저 참조한 후 1을 더함
    int result2 = *ary+ 1;     // 위와 완전히 동일 (공백 무시)
    int result3 = *(ary + 1);  // 주소를 1칸(int 크기) 이동한 후 값을 참조

    printf("*ary + 1 의 결과: %d\n", result1); // 10 + 1 = 11
    printf("*ary+ 1 의 결과: %d\n", result2);  // 10 + 1 = 11
    printf("*(ary + 1) 의 결과: %d\n", result3); // ary[1]의 값 = 20

    return 0;
}
```

## 배열명 = 시작 주소

C 언어에서 배열의 이름(예: `ary`)은 배열의 첫 번째 요소(`&ary[0]`)를 가리키는 포인터 상수입니다.
때문에 `ary`와 `&ary[0]`는 같은 주소값을 담고있습니다.

## 배열과 포인터 표기법 호환

`ary[1]`과 `*(ary + 1)`은 컴파일러 입장에서 완벽히 동일한 코드입니다.
> `ary[1]`은 배열에서 인덱스가 1인 요소의 값
> `*(ary + 1)`은 `배열의 첫번째 요소 + 1`번째 요소의 값

## 🚨 연산자 우선순위와 공백

`*ary + 1;`과 `*ary+ 1;`처럼 띄어쓰기가 어떻게 되어 있든 컴파일러는 공백을 무시합니다.
포인터 간접 참조 연산자(`*`)가 산술 연산자(`+`)보다 우선순위가 높기 때문에, 무조건 배열 첫 번째 값(`*ary)`을 먼저 읽어온 후 숫자 `1`을 더하게 됩니다.

주소값을 먼저 연산하고 싶다면 반드시 `*(ary + 1)`처럼 괄호를 사용해야 합니다.

## 한눈에 정리하기
| 표기 | 포멧 | 내용 | 동일한 표현 |
| -- | -- | -- | -- |
| `ary` | 주소 | 배열의 시작 주소 | `&ary[0]` |
| `ary + 1` | 주소 | 두 번째 요소의 주소 | `&ary[1]` |
| `&ary[0]` | 주소 | 인덱스가 0인 요소의 주소 | `ary` |
| `ary[0]` | 값 | 인덱스가 0인 요소의 값 | `*ary` |
| `ary[1]` | 값 | 인덱스가 1인 요소의 값 | `*(ary + 1)` |

* **`&`나 그냥 이름(`ary`)**만 있으면 무조건 **'주소(위치)'**를 의미합니다.
* **`*`나 괄호 대괄호(`[]`)**가 붙어 있으면 무조건 껍데기를 깐 **'실제 값(알맹이)'**을 의미합니다.
* `ary + 1`은 단순한 숫자 1을 더하는 것이 아니라, 다음 배열 칸으로 '1칸 점프'하라는 뜻입니다.
