파이썬에서 변수는 상자가 아니라 **'이름표(주소)'**입니다. `a = [1, 2, 3]`을 만들고 `b = a`를 하면, 리스트가 2개가 되는 것이 아니라 하나의 리스트에 `a`와 `b`라는 두 개의 이름표가 붙을 뿐입니다. 따라서 `b`를 수정하면 `a`도 같이 변해버립니다.

이 문제를 해결하기 위해 **슬라이싱(`[:]`)**을 사용하여 리스트를 복사하곤 하는데, 출제위원들은 바로 이 지점에 **'2차원 리스트의 얕은 복사 함정'**을 파놓습니다.

---

## 1. 1차원 리스트의 복사 (문제없음)

단순한 1차원 리스트는 슬라이싱(`[:]`)이나 `.copy()`를 쓰면 완벽하게 독립적인 새로운 리스트로 복사됩니다.

```python
a = [1, 2, 3]
b = a[:]      # 처음부터 끝까지 슬라이싱하여 새로운 리스트 생성 (복사)

b[0] = 99     # 복사본 b만 수정

print("원본 a:", a) # 출력: [1, 2, 3] (안전함!)
print("복사본 b:", b) # 출력: [99, 2, 3]
```

## 2. 🚨 2차원 리스트의 얕은 복사 함정 (26년 1회차 기출 원리)

리스트 안에 리스트가 들어있는 2차원 구조일 때 슬라이싱(`[:]`)을 하면, '바깥쪽 껍데기'만 새로 만들어지고, '안쪽 알맹이(내부 리스트)'는 여전히 원본과 같은 메모리 주소를 공유하게 됩니다. 이것을 얕은 복사(Shallow Copy)라고 합니다.

```Python
# 2차원 리스트 생성
m = [[1], [2], [3]]
b = m[:]  # 얕은 복사 실행 (껍데기만 새로 만들어짐)

# 🚨 복사본 b의 내부 리스트 값을 수정해 봅니다.
b[0][0] = 99 

# 원본 m도 같이 수정되어 버립니다!
print("원본 m:", m) # 출력: [[99], [2], [3]]
print("복사본 b:", b) # 출력: [[99], [2], [3]]
```

### 💡 왜 이런 일이 발생할까?

`b = m[:]`를 하는 순간, `m`과 `b`라는 서로 다른 출입문이 생긴 것입니다.
하지만 다른 문으로 들어가도 결국 그 안에 들어있는 내용물(내부 리스트 [1], [2])은 여전히 똑같습니다. 
결국 b 문을 열어서 첫 번째 링크(b[0])를 타고 들어가 내용을 99로 수정하면, 똑같은 링크 주소를 공유하고 있던 m 문을 열어서 들어가도 내용이 99로 바뀌어 있는 것입니다.

## 3. 완벽한 복제를 원한다면: 깊은 복사 (Deep Copy)

내부 리스트까지 뼛속까지 완벽하게 독립적인 클론을 만들려면, 파이썬 내장 모듈인 copy의 `deepcopy()` 함수를 사용해야 합니다.

```Python
import copy

m = [[1], [2], [3]]
b = copy.deepcopy(m) # 내부 리스트까지 완벽하게 새로 생성

b[0][0] = 99

print("원본 m:", m) # 출력: [[1], [2], [3]] (원본 방어 성공!)
print("복사본 b:", b) # 출력: [[99], [2], [3]]
```
