파이썬의 4대 컬렉션 중 리스트와 튜플이 '순서'가 있는 기차라면, **딕셔너리(Dict)**는 이름표가 붙은 사물함이고, **세트(Set)**는 중복을 허용하지 않는 마법의 주머니입니다. 최근 정보처리기사 실기 시험에서는 이 두 가지 자료구조를 자유자재로 다루고, 나아가 `lambda` 함수를 이용해 원하는 기준으로 정렬하는 능력을 강하게 요구하고 있습니다.

## 1. 딕셔너리(`Dict`) 심화: 순회와 안전한 접근

딕셔너리는 `Key: Value` 쌍으로 이루어져 있습니다. `for`문으로 딕셔너리를 돌릴 때, 딕셔너리가 기본적으로 무엇을 뱉어내는지 정확히 알아야 합니다.

```python
scores = {'Kim': 80, 'Lee': 95, 'Park': 70}

# 🚨 함정 1: 그냥 딕셔너리를 for문에 넣으면 'Key(이름표)'만 튀어나옵니다.
for x in scores:
    print(x) # 출력: Kim, Lee, Park

# ✅ 정석 1: Key와 Value를 동시에 뽑으려면 무조건 .items()를 씁니다.
for k, v in scores.items():
    print(f"이름: {k}, 점수: {v}")

# ✅ 정석 2: 에러 없이 값 가져오기 (.get)
# scores['Choi']를 하면 에러가 나지만, get()을 쓰면 안전합니다.
print(scores.get('Choi', 0)) # Choi가 없으면 기본값 0을 반환하라
```

## 2. 세트(`Set`) 심화: 중복 제거와 집합 연산

세트는 수학의 '집합'과 완벽히 똑같습니다. 2023년 1회차, 2025년 2회차에 걸쳐 꾸준히 출제되는 빈출 주제입니다.

```Python
list_data = [1, 2, 2, 3, 3, 3]

# 1. 마법의 중복 제거 (리스트를 세트로 변환)
unique_data = set(list_data)
print(unique_data) # 출력: {1, 2, 3}

# 2. 집합 연산 (교집합, 합집합, 차집합)
a = {1, 2, 3}
b = {3, 4, 5}

print(a & b) # 교집합 출력: {3}
print(a | b) # 합집합 출력: {1, 2, 3, 4, 5}
print(a - b) # 차집합 출력: {1, 2}
```

## 3. `lambda`를 활용한 정렬: `sorted()`

리스트나 딕셔너리를 정렬할 때 쓰는 `sorted()` 함수에 `lambda`(이름 없는 1회용 함수)를 결합하면, 내가 원하는 아주 복잡한 기준으로 데이터를 정렬할 수 있습니다.
**📌 기본 공식: `sorted(데이터, key=lambda 변수: 정렬기준)`**

```Python
# [상황 1] 2차원 리스트 정렬
data = [[1, 5], [3, 2], [2, 8]]

# 기본 정렬: 앞의 숫자(1, 3, 2)를 기준으로 오름차순 정렬됨
print(sorted(data)) # 출력: [[1, 5], [2, 8], [3, 2]]

# 🚨 lambda 활용: "뒤에 있는 숫자(인덱스 1)를 기준으로 정렬해라!"
# x는 리스트 안의 요소 하나하나([1,5] 등)를 의미하고, x[1]은 그 요소의 두 번째 값을 뜻합니다.
sorted_data = sorted(data, key=lambda x: x[1])
print(sorted_data) # 출력: [[3, 2], [1, 5], [2, 8]] 


# [상황 2] 딕셔너리를 Value(점수) 기준으로 정렬하기
scores = {'Kim': 80, 'Lee': 95, 'Park': 70}

# 딕셔너리에 .items()를 쓰면 ('Kim', 80) 같은 튜플 형태로 나옵니다.
# x[0]은 Key(이름), x[1]은 Value(점수)입니다.
# 점수(x[1])를 기준으로 내림차순(reverse=True) 정렬!
rank = sorted(scores.items(), key=lambda x: x[1], reverse=True)

print(rank) 
# 출력: [('Lee', 95), ('Kim', 80), ('Park', 70)]
```
