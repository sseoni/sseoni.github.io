## 1. 튜플(Tuple)의 탄생과 불변성

튜플은 소괄호 `()`를 사용하여 만든다. 리스트와 달리 **한 번 만들어지면 그 안의 값을 절대 바꾸거나 지울 수 없는 불변성(Immutability)**을 가진다.

위도와 경도, RGB 색상 코드처럼 프로그램이 실행되는 동안 실수로라도 데이터가 변경되면 안 되는 중요한 정보를 담을 때 주로 사용힌다.

```python
# 튜플 만들기
rgb_color = (255, 0, 0) # 빨간색 코드

# 인덱싱으로 값을 읽는 것은 가능
print(rgb_color[0]) # 출력: 255

# 🚨 값을 변경하려고 하면 에러 발생
# rgb_color[0] = 0  -> TypeError 발생!
```

**💡 튜플 생성 시 주의점**
데이터가 딱 1개뿐인 튜플을 만들 때는 반드시 숫자 뒤에 콤마(,)를 찍어주어야 한다. 그렇지 않으면 파이썬은 이를 튜플이 아닌 단순 괄호(수학 연산용)로 인식한다.

```python
single_tuple = (1,) # 정상적인 튜플
not_tuple = (1) # 단순 정수(int) 1로 인식
```


## 2. 튜플의 유용한 메소드

수정이 불가능하므로 append()나 remove() 같은 기능은 없지만, 데이터를 확인하는 기능은 사용할 수 있다.

`.count(값)`: 튜플 안에 해당 값이 몇 개 있는지 센다.

`.index(값)`: 해당 값이 처음 등장하는 위치(인덱스)를 알려준다.


## 3. 패킹(Packing)과 언패킹(Unpacking)

튜플을 사용하는 가장 큰 이유 중 하나다. 여러 개의 데이터를 하나의 변수로 묶는 것을 패킹, 묶여 있는 데이터를 다시 여러 개의 변수로 풀어헤치는 것을 언패킹이라고 한다.

괄호를 생략해도 튜플로 묶이기 때문에 코드를 매우 간결하게 짤 수 있다.

```Python
# 1. 튜플 패킹 (소괄호 생략 가능)
# 1, 2, 3이 튜플 (1, 2, 3)으로 묶여 my_tuple 변수에 들어갑니다.
my_tuple = 1, 2, 3 

# 2. 튜플 언패킹
# 튜플 안의 값들이 각각 a, b, c 변수에 흩어져서 들어갑니다.
a, b, c = my_tuple 
print(a) # 출력: 1
print(b) # 출력: 2
print(c) # 출력: 3

# 💡 언패킹을 활용한 변수 값 교환 (Swap)
# 타 언어처럼 임시 변수(temp)를 만들 필요 없이 한 줄로 두 변수의 값을 바꿀 수 있습니다.
x = 10
y = 20
x, y = y, x # (x, y) = (20, 10) 으로 언패킹되어 값이 교환됨
```


## 4. 별표(`*`)를 활용한 확장된 언패킹 (Extended Unpacking)

튜플 안의 데이터 개수가 많거나 정확히 알 수 없을 때, 특정 데이터만 변수에 담고 나머지는 하나의 **리스트**로 묶어서(`*others` 등) 받을 수 있다.
묶어주고 싶은 변수 이름 앞에 별표(`*`)를 붙이면 된다.

**❗️ 주의**: 직접 담은 변수 이외의 others는 리스트로 저장된다.

```python
# [확장된 언패킹 예제]
scores = (100, 95, 90, 85, 80)

# 첫 번째 값만 1등(first) 변수에 넣고, 나머지는 모두 모아서 others 리스트에 넣습니다.
first, *others = scores

print(first)  # 출력: 100 (정수)
print(others) # 출력: [95, 90, 85, 80] (나머지 값들이 리스트 형태로 묶임)

# 💡 응용: 양 끝의 데이터만 빼고 가운데 데이터들만 묶을 수도 있습니다.
first, *middle, last = scores
print(middle) # 출력: [95, 90, 85]
```
