파이썬 실기 시험에서 함수 문제가 나왔을 때, 매개변수 자리에 `def func(a, lst=[]):` 처럼 **리스트가 기본값으로 자리 잡고 있다면 100% 함정 문제**입니다. 이 원리를 모르면 코드를 눈으로 따라가다가 완전히 길을 잃게 됩니다.

---

## 1. 치명적 함정: 리스트 기본값의 배신

함수를 정의할 때 매개변수의 기본값을 빈 리스트(`[]`)로 설정해 보겠습니다. 우리의 상식으로는 함수를 호출할 때마다 새로운 빈 리스트가 짠! 하고 나타나야 할 것 같습니다.

```python
# lst=[] 처럼 가변 객체(리스트, 딕셔너리 등)를 기본값으로 주면 끔찍한 일이 발생합니다.
def append_num(n, lst=[]):
    lst.append(n)
    return lst

# 1. 첫 번째 호출 (정상 작동하는 것처럼 보임)
r1 = append_num(1)
print(r1) # 출력: [1]

# 2. 두 번째 호출 🚨 (빈 리스트에서 시작하지 않음!)
r2 = append_num(2)
print(r2) # 출력: [1, 2]

# 3. 세 번째 호출 🚨 (계속 누적됨!)
r3 = append_num(3)
print(r1) # 출력: [1, 2, 3]
print(r3) # 출력: [1, 2, 3]
```

## 2. 출제 원리: 함수 기본값은 '단 한 번만' 만들어진다.

파이썬에서 **변수**는 데이터를 담는 공간이 아니라, 객체를 가리키는 **'이름표(참조)'**일 뿐입니다.

파이썬에서 함수의 기본값(Default Argument)은 함수가 매번 호출될 때마다 새로 만들어지는 것이 아닙니다. 프로그램이 처음 실행되면서 함수를 메모리에 읽어 들일 때(정의될 때) 딱 한 번만 만들어집니다.

**👉🏻 `def append_num(n, lst=[]):`**

- 코드를 파이썬이 읽는 순간, 메모리 어딘가에 `lst`라는 이름의 빈 리스트 `[]`를 딱 1개 생성하고,
- 함수는 이 객체의 메모리 주소를 영구적으로 기억합니다.

**👉🏻 `append_num(1)` 호출 ➔ 그 유일한 리스트에 1이 추가됩니다.**

- 지역 변수 이름표 `lst`가 생성되어 방금 그 주소를 가리킵니다.
- 그리고 `.append(1)`을 통해 리스트 객체 내부에 직접 접근해 값 1을 집어넣습니다.

**👉🏻 `append_num(2)` 호출 ➔ 그 유일한 리스트에 2가 추가됩니다.**

- 이름표 lst는 또다시 함수가 기억하고 있던 '같은 주소'를 가리킵니다.
- 그래서 새로운 리스트를 만들지 않고, 아까 개업해서 계속 쓰고 있던 그 리스트를 다시 가져옵니다.
- 때문에 기존에 있던 1 옆에 2가 추가되어 `lst`는 `[1, 2]`가 됩니다.

이처럼 리스트(`List`), 딕셔너리(`Dict`), 세트(`Set`) 같이 안의 내용을 바꿀 수 있는 **가변 객체(Mutable)**를 기본값으로 쓰면 값이 계속 누적되는 함정이 완성됩니다.

## [참고 1] 매번 새로운 빈 리스트 생성하기

이 함정을 피하고 매번 새로운 빈 리스트를 생성하려면, 기본값을 None으로 주고 함수 내부에서 지역 변수 리스트를 새로 할당하도록 코드를 짜야 합니다.

```Python
def safe_append_num(n, lst=None):
    # 호출될 때마다 새롭게 빈 리스트를 생성
    if lst is None:
        lst = []
    
    lst.append(n)
    return lst

print(safe_append_num(1)) # 출력: [1]
print(safe_append_num(2)) # 출력: [2] (정상 작동!)
```

## [참고 2] '불변 객체(정수, 문자열 등)'라면 어떻게 될까?

```python
def count_up(n=0): # 정수(int)는 불변 객체!
    n += 1
    return n

print(count_up()) # 출력: 1
print(count_up()) # 출력: 1 (누적 안 됨!)
```

이 경우는 왜 누적이 되지 않을까요? 핵심은 정수(`int`)가 **'불변(Immutable)'** 객체라는 점입니다.
함수 정의 시 `0`이라는 정수 객체의 주소가 함수의 기본값으로 영구 등록됩니다.
`n += 1` 연산을 만나는 순간, 기존의 `0` 객체를 `1`로 뜯어고치는 것은 불가능합니다 (불변의 법칙).
따라서 파이썬은 메모리에 완전히 새로운 `1` 객체를 생성한 후, 지역 변수 이름표 `n`을 떼어서 새로운 `1` 객체에 붙여버립니다 (재할당).
결과적으로 함수의 기본값이 원래 기억하고 있던 `0` 객체는 아무런 영향을 받지 않고 보존되므로, 다음 호출 때도 어김없이 0부터 시작하게 되어 누적이 발생하지 않습니다.

### 💡 Tip
시험 문제에 `def func(lst=[]):` 가 보인다면, 함수가 여러 번 호출되더라도 메모리(백지)에 네모 박스를 딱 하나만 그리고 그곳에 값을 계속 누적해서 추적하세요!
