2025년 1회차 실기 시험 파이썬 파트에서 수험생들을 절망에 빠뜨린 전설의 문제가 있습니다. 바로 **클래스(Class)를 이용해 트리(Tree) 자료구조를 직접 만들고, 재귀 함수로 이를 순회하는 문제**입니다. 

이 문제는 과거 파이썬 파트에서 묻던 단순한 문자열, 리스트 제어를 넘어섰습니다. 파이썬이 완벽한 **객체 지향 프로그래밍(OOP) 언어**임을 인지하고, 클래스를 이용해 노드(Node) 객체를 생성하여 자료구조를 직접 엮어낼 수 있는지를 묻는 고차원적인 문제입니다. 그 핵심 동작 원리를 완벽하게 해체해 봅시다.

---

## 1. 파이썬 클래스(Class)로 노드(Node) 설계하기

트리 구조를 만들려면 데이터를 담고 서로를 연결할 '노드' 객체가 필요합니다. 파이썬에서는 이를 **클래스(Class)**로 정의합니다.

```python
# 1. 파이썬의 Node 클래스 정의
class Node:
    # __init__ 은 객체가 생성될 때 무조건 실행되는 초기화(생성자) 메서드입니다.
    def __init__(self, value):
        self.value = value      # 내 데이터(알맹이)를 저장하는 속성
        self.children = []      # 내 자식 노드들의 '참조(메모리 주소)'를 담을 빈 리스트
```

`self`의 의미: 만들어진 객체 인스턴스 자기 자신을 가리키는 파이썬의 고유 키워드입니다. Java의 `this`와 같은 역할을 합니다.
`self.children = []:` 파이썬의 리스트는 가변 객체(Mutable)이므로, 크기 제한 없이 다른 노드 객체의 주소(참조)를 무한히 담을 수 있습니다. 이를 활용해 부모와 자식 노드를 연결합니다.

## 2. 2025년 1회차 기출 분석

실제 기출문제의 코드를 단계별로 추적해 보겠습니다.

### 📌 [Step 1] 리스트 데이터를 트리(Tree) 객체로 엮기

부모 노드와 자식 노드의 인덱스 수학 공식을 이용해, 평면적인 리스트의 값들을 입체적인 트리 구조로 엮어내는 과정입니다.

```Python
def tree(li):
    # 1. 컴프리헨션: 리스트 li의 모든 숫자를 각각 독립된 Node 객체로 변환합니다.
    nodes = [Node(i) for i in li] 
    
    # 2. 부모-자식 객체 연결 로직
    for i in range(1, len(li)):
        # 2진 트리의 수학적 배열 성질: 인덱스 i 노드의 부모 인덱스는 항상 (i - 1) // 2 입니다.
        parent_index = (i - 1) // 2
        
        # 부모 객체의 children 리스트에 자식 객체의 참조(주소)를 추가(.append)합니다.
        nodes[parent_index].children.append(nodes[i])
        
    return nodes[0] # 트리의 최상단(Root) 노드 객체를 반환

li = [3, 5, 8, 12, 15, 18, 21]
root = tree(li)
```

**[객체 참조 연결 상태 확인]**
`root (인덱스 0)`: 값 3 / 자식: 노드(5), 노드(8)
`인덱스 1 (값 5)`: 자식: 노드(12), 노드(15)
`인덱스 2 (값 8)`: 자식: 노드(18), 노드(21)
➔ 전형적인 2진 트리(Binary Tree) 계층 구조가 메모리 상에 완성되었습니다.

### 📌 [Step 2] 재귀 함수로 트리 순회하기 (핵심 킬러 로직)

트리의 Root 노드부터 시작하여, 조건에 맞는 값만 취사선택하여 누적 합산하는 재귀 함수입니다.

```Python
def calc(node, level=0):
    # 탈출 조건: 탐색할 노드 객체가 없으면 0 반환
    if node is None:
        return 0
        
    # 현재 노드의 값 결정 로직: 홀수 레벨(level % 2 == 1)일 때만 자기 값을 취하고, 짝수면 0을 취함
    current_val = node.value if level % 2 == 1 else 0
    
    # 재귀 호출: 내 자식 노드(children 리스트)들을 하나씩 꺼내어 레벨을 1 올린 뒤 다시 calc() 함수로 탐색
    child_sum = sum(calc(n, level + 1) for n in node.children)
    
    # 내 값과 자식들의 합산 결과를 더해서 호출자에게 반환
    return current_val + child_sum

print(calc(root)) 
```

### 💡 실전 디버깅 (손으로 추적하기)

이런 재귀 트리를 만났을 때는 시험지 여백에 트리를 그려놓고 계층(Level)을 층별로 나누어 값을 추적하는 것이 가장 정확합니다.

Level 0 (루트): 객체 값 `3`
 - ➔ 짝수 레벨이므로 `current_val = 0`

Level 1 (자식 1층): 객체 값 `5`, `8`
 - ➔ 홀수 레벨이므로 `current_val = 5, 8`

Level 2 (자식 2층): 객체 값 `12`, `15`, `18`, `21`
 - ➔ 짝수 레벨이므로 모두 `current_val = 0`

🧮 최종 합산:
`calc()` 함수는 모든 트리 객체를 훑으면서 값을 반환합니다.
0 (루트) + 5 (1층) + 8 (1층) + 0 (2층 전부) = 13
